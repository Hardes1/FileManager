Приветствую вас в файловом менеджере. Хотел бы тут написать несколько моментов касательно взаимодействия.

1) Не допускается произвольное количество символов между двумя командами (т.е нельзя написать "cd          ../")
2) Приложения старается быть похожим на unix terminal.
3) По-мнению автора, задания 9-10 базового ТЗ несколько бессмысленны (зачем создавать пустой файл в конкретной кодировке???), поэтому он немного усложнил
для себя задачу и добавил возможность записывать текст в этот файл при его создании, однако для этого каждый раз нужно указывать явным образом кодировку из списка доступных.
например, я хочу создать в текущей директории файл mytext.txt и записать в него текст "Hello world" в кодировке ascii. Тогда для этого нужно написать следующее:
"create mytext.txt ascii Hello world" (т.е не сработает написать "create mytext.txt Hello world"). Для того, чтобы просто создать файл в соответствующей кодировке, досаточно
написать "create mytext.txt [your encoding]", где [your encoding] одна из доступных кодировок, обратите внимание, что этот параметр не является обязательным, то есть можно написать лишь
"create mytext.txt" - создастся файл в кодировке utf-8 (она стоит по умолчанию). Для перевода строки используйте символ '\n' пример: "hello\nworld" 
4) Во время таких действий как перемещение файла, копирование файла, конкатенация файлов, а так же создание нового файла (и запись в него чего-то) создают новый файл на месте целевой директории, а если
такой файл там уже есть, то они его перезаписывают.
5) Для смены директории и диска работает команда "cd". Если вы, например, находитесь на диске C: сейчас и хотите поменять его на диск D, то вам нужно написать: "cd D:"
6) Команда cat (объединить все файлы в один) сработает, если есть хотя бы два аргумента (нельзя написать "cat destination.txt"). 
7) Команда diff показывает необходимые действия, чтобы превратить левый файл, в правый, для этого достаточно указать их путь (можете для примера написать "diff a.txt b.txt").
По желанию результат можно записать в другой файл. "+" означает, что эту строку нужно добавить, "=" означает, что эта строка встречается в обоих файлах и её добавлять не нужно,
 "-" означает, что текущую строку надо удалить.
8) для работ с файлами всегда нужно явно указывать расширение файла (т.е. если у вас есть файл hello.txt в текущей директории, то нельзя написать "read hello", нужно
писать "read hello.txt").
